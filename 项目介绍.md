# JobSeekHelper项目介绍

## 项目背景

当前，大学生求职越来越成为全社会关注的问题。在实际中，大学生找工作/实习时经常存在以下问题：

1. 简历不过关：简历中的内容和岗位的匹配度不强，经常写了很多但是关键的很少；

2. 面试考点不了解：对于当下行业里面试时的常见考点不知道，无法有针对性的准备；

3. 项目经历不熟悉：一些课程项目/科研项目/实习项目虽然完成了，但是对于一些底层架构、技术栈选取、具体的算法等一些细节掌握不深入，在面试时容易出错。

## 项目功能

本项目主要要实现以下功能：

1. 简历内容上传与输入：用户可以上传自己的简历（支持pdf、docx、pptx、txt格式的文档），系统自动解析其中的各部分内容（包括用户的个人基本信息、学历信息、项目经历、实习经历、获奖经历、外语水平等）；用户也可以在个人主页中直接输入各部分的内容。

2. 简历生成：根据用户所上传的个人信息的各部分内容，系统可以根据用户要应聘的岗位，从中选取匹配度高的项目/实习经历，并从中提取相关内容作为“在项目中承担的工作”，最终生成一份内容针对性强、排版布局美观的简历（PDF版本）。生成简历时可调用LLM的API。

3. 专业知识上传与RAG解析：用户可以上传与自己应聘的岗位相关的知识文档（例如，对于后端开发的求职者来说，他可以上传与后端开发相关的文档，包括JVM、数据库、Redis等常见考点）。文档上传后，系统会进行RAG检索，以供“模拟面试对话”功能所用。

4. 模拟面试对话：这是本系统最重要的功能。系统根据用户所要应聘的岗位和个人信息，调用LLM联网查询当下常见的面试问题并生成面试模拟考题。模拟考题主要包含两部分内容：行业中该类型的岗位常见的问题（例如，对于后端开发来说，包括JVM的内存管理、数据库的索引等基础知识），用户的项目经历/实习经历的具体细节。在面试时，既要在屏幕上显示考题，又要语音播报；用户在回答问题时要实时进行语音转文字，然后经用户检查核对识别无误后再发送，要支持用户对语音转出来的文字进行编辑。可以由LLM决定什么时候结束面试（评判标准：面试时长超过40分钟，且面试过程中涵盖了大部分的知识点），也可以由用户决定什么时候结束面试（界面中有一个“结束面试”的按钮，用户点击后即结束）。面试结束后，系统调用LLM对用户的回答进行评价，评价时需要根据“功能点3”的RAG的结果来准确评估用户的回答是否正确、什么地方出错，如果在RAG的资料中没有找到有关内容，也可以联网查询；对于一些用户个人项目经历/实习经历的问题，需要给出有待加深理解和提高的地方。

5. 支持多岗位：系统要支持用户应聘多个岗位，如后端开发、前端开发、AI工程师、产品经理等。对于不同的岗位，用户可以上传不同的技术知识文档，系统也应根据用户的个人信息和项目经理，生成有针对性的个人简历和模拟面试考题。

6. 防作弊检测：当模拟面试开始后，要确保系统界面全屏显示（也就是说屏幕上不能有其他内容），同时一旦检测到用户切换屏幕，面试自动结束。
